How to use this page

Keep each project card concise.

For each project, include a title, a one- or two-sentence summary, a few tags, and a link to a detailed project page. That gives visitors enough context without making the overview page too dense.

Project page structure

Suggested detail-page sections

  • Project title and short overview
  • Your role and the tools you used
  • Process, steps, or workflow
  • Images, screenshots, or outputs
  • Main takeaway or what you learned
Good practice

Keep the project story focused.

Even when a project is technical, it helps to explain the purpose first, then the process, then the result. That makes the page readable for classmates, instructors, and future visitors.
