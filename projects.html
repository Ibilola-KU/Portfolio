Custom GPT project

This project focuses on a practical documentation task in construction administration.

Project: Construction RFI Analyst

What it is: A custom mini-agent built to help users review, summarize, classify, draft, and improve construction Requests for Information (RFIs).

What it represents: This project demonstrates how AI can support construction documentation workflows by extracting key information, identifying weak or missing details, and improving the clarity of project communication.

Why it matters: RFIs are a routine but time-sensitive part of project delivery. A focused AI tool can help reduce repetitive review work, improve the quality of written RFIs, and make requests easier for design and project teams to answer.

How it was built

The mini-agent was intentionally scoped only to RFIs so that its responses would stay focused and consistent. Its instructions defined five main tasks: reviewing existing RFIs, summarizing the issue, classifying the likely RFI type, identifying missing or weak information, and drafting new RFIs from field issues.

It was also instructed not to invent project facts, drawing references, specification sections, cost impacts, or schedule impacts when that information was missing.

Knowledge and references used

The setup was supported by three main references: an Autodesk Construction Cloud RFI template, a set of sample RFIs, and an RFI writing guide created for this project.

Those materials were used to ground the GPT in standard RFI structure, common RFI categories, writing principles, and common weaknesses such as vague wording, missing references, and unclear locations.

What I learned

Testing showed that the mini-agent was most useful for repetitive tasks such as reviewing wording, flagging unclear references or missing details, and improving vague draft RFIs. It was especially helpful when the goal was to make an RFI easier to understand and easier to answer.

Limits and professional judgment

The project also showed that AI still needs human review. In testing, the mini-agent could become repetitive and sometimes treated placeholder text as if it were real project information. It could improve wording, but it could not recover missing project-specific details such as exact locations or drawing references that were never provided.

That reinforced an important lesson: AI can be useful as a support tool in construction documentation, but it is not a substitute for project knowledge, responsible review, or professional judgment.
